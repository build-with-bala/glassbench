# GlassBench baselines

Reference systems that produce a `submissions/<name>/predictions.json` for GlassBench.
There are two kinds here:

1. **Cached, key-free baselines** — deterministic Python, no LLM, no API key. They give
   the leaderboard its floors and ceilings. Re-running them reproduces byte-identical
   output.
2. **A generic LLM adapter** (`llm_adapter.py`) — turns *any* chat-LLM (Anthropic or
   OpenAI) into a GlassBench system. Needs an API key; if none is set it prints a clear
   message and exits **without fabricating anything**.

Every baseline reads only each item's `id`, `history`, and `query`. None reads
`gold_answer`, `evidence_spans`, `answer_topic`, or `split` at inference time, so all are
valid under [`CONTRIBUTING.md`](../CONTRIBUTING.md). The output format is the frozen
contract from [`PRE_REGISTRATION.md`](../PRE_REGISTRATION.md):

```jsonc
{"id": "<item id>", "answer": "Seattle", "confidence": 0.91}   // answered
{"id": "<item id>", "abstain": true}                            // abstained
```

The interpreter used in development is plain CPython with **numpy** only. The cached
baselines need nothing else. The LLM adapter additionally needs the relevant provider
SDK (`anthropic` or `openai`) installed in whatever interpreter you run it with.

---

## (a) Reproduce the cached, key-free baselines (no API key)

Run any of these from the repo root. Each writes `submissions/<name>/predictions.json`.

```bash
python baselines/always_answer.py        # answers everything (recency); high CWR floor
python baselines/always_abstain.py       # abstains everywhere; collapses on answerable
python baselines/bm25_retrieval.py       # retrieves best history sentence; score->conf
python baselines/random_confidence.py    # answers all, uniform-random conf (calib floor)
python baselines/agent_llm.py            # genuine imperfect abstention-aware agent
```

These are **deterministic**: no RNG seed needed (`random_confidence` uses a fixed seed
internally), no network, no key. Two runs are byte-identical.

> `verbalized_confidence_llm` and `abstention_aware_llm` are **constructed references**
> (their behaviour is keyed to the gold split) and are shipped as cached
> `submissions/.../predictions.json` only — they are intentionally *not* regenerated by a
> script and are excluded from the headline ranking. See their `system.md`.

Score any of them with the same scorer everyone uses:

```bash
python -m glassbench.score --predictions submissions/agent_llm/predictions.json
```

## (b) Run the adapter with a key to score a new LLM

`llm_adapter.py` sends each item's conversation `history` + `query` to a chat model and
asks for an answer + a calibrated confidence, **or** an abstention (when the fact was
retracted/updated, was never stated, or the question rests on a false premise). It parses
the model's JSON reply into the submission contract and writes both
`submissions/<name>/predictions.json` and a `submissions/<name>/system.md` stub.

### 1. Install a provider SDK and set its key

```bash
# Anthropic:
pip install anthropic
export ANTHROPIC_API_KEY=sk-ant-...

# …or OpenAI:
pip install openai
export OPENAI_API_KEY=sk-...
```

The provider is **auto-detected** from whichever key is set. If neither is set the script
prints what to do and exits with code 2 — it never invents predictions.

### 2. Smoke-test on a few items first (cheap)

```bash
python baselines/llm_adapter.py --name my-llm --limit 5
```

You'll see per-item progress (`answer (conf=…)` or `ABSTAIN`).

### 3. Run the full set

```bash
python baselines/llm_adapter.py --name my-llm
```

Writes `submissions/my-llm/predictions.json` (+ `system.md`) over all 96 items.

### 4. Score it

```bash
python -m glassbench.score --predictions submissions/my-llm/predictions.json
```

### Options

| Flag | Default | Meaning |
|------|---------|---------|
| `--name` | `<provider>-<model>` slug | submission folder name (the leaderboard row name) |
| `--provider` | auto-detect from env | `anthropic` or `openai` (force one) |
| `--model` | provider default (see below) | any current model id; overrides `*_MODEL` env |
| `--limit N` | all 96 | only run the first N items (smoke test) |
| `--max-tokens` | 512 | max output tokens per call |
| `--max-history-chars` | 24000 | truncate the oldest turns of long transcripts |
| `--retries` | 3 | retries per item on transient API errors |
| `--quiet` | off | suppress per-item progress lines |

**Model defaults** (override with `--model`, `ANTHROPIC_MODEL`, or `OPENAI_MODEL` so a
stale default never blocks you):

- Anthropic: `claude-sonnet-4-0`
- OpenAI: `gpt-4o-mini`

Example forcing a specific model and naming the row:

```bash
python baselines/llm_adapter.py \
  --provider anthropic --model claude-sonnet-4-0 --name claude-sonnet-4
```

### How the adapter behaves (so you can trust the numbers)

- **Reads no labels.** The prompt is built from `history` + `query` only. (Proof: it
  builds an identical prompt even if every label field is stripped from the item.)
- **Confidence is the model's own.** It uses the `confidence` field the model returns,
  clamped to `[0, 1]`. It does not post-hoc calibrate — what you submit is the model's
  raw verbalized confidence, which is exactly what GlassBench is built to measure.
- **Conservative on failure.** If the reply has no parseable answer, or all retries fail,
  the item is recorded as `abstain` rather than a fabricated guess. This never invents a
  *confident* wrong answer, but note it does shift those items toward the abstention
  side — keep an eye on the per-item log for repeated parse/API failures.
- **Not byte-deterministic.** Unlike the cached baselines, LLM output varies run to run;
  re-running may move a few predictions. The generated `system.md` records the exact
  model id used for the run.

### Adding another provider

Subclass `Provider` in `llm_adapter.py` (implement `complete(system, user) -> str`),
detect its key in `select_provider`, and add it to the `--provider` choices. The prompt,
parser, run loop, and writers are provider-agnostic and unchanged.
