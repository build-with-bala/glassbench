# GlassBench public site

A self-contained, **static** site for the GlassBench benchmark — an interactive
**Overview**, a sortable/filterable **Leaderboard**, and the **Datasheet**. It is
presentation only: it never imports or runs the `glassbench` package, the scorer, or
the data, and it recomputes no scores.

The numbers shown are the verified-live values that also appear verbatim in the repo's
`LEADERBOARD.md` (the deterministic output of `scripts/gen_leaderboard.py`). The
leaderboard data is held once as a Python literal inside `build_site.py` and emitted to
`leaderboard.json`, so the site can never disagree with the repo.

The site stays **dependency-light and deploys unchanged**: no pip install, no node, no
bundler. Pages and the data file are generated by a stdlib-only Python script; the only
runtime dependency is the ECharts chart library, loaded from a CDN `<script src>`. With
JavaScript or the CDN unavailable the site still works — the leaderboard table is
server-rendered, charts fall back to visually-hidden data tables, and the confidence
dial falls back to a static results table.

## Files

| Path | What it is |
|---|---|
| `build_site.py` | Stdlib-only generator. Holds the verified leaderboard data, renders the hand-authored Overview + Leaderboard, renders the Datasheet from `DATASHEET.md`, and writes `style.css`, `app.js`, and `leaderboard.json`. No pip install needed. |
| `nginx.conf` | nginx server block (extensionless URLs, caching, basic hardening, gzip incl. JSON/JS). |
| `dist/` | Generated output snapshot: `index.html`, `leaderboard.html`, `datasheet.html`, `style.css`, `app.js`, `leaderboard.json`. Convenience only — the Docker build regenerates it. |

### What the generator emits into the output dir

- `index.html` — hand-authored interactive Overview (hero CWR strip, four-split matrix,
  Glass Score schematic, the "Try to game it" confidence dial, trust band, methodology).
  Preserves the README anchor slugs other docs link to.
- `leaderboard.html` — hand-authored shell + a **server-rendered** static leaderboard
  table (works with JS off), enhanced by `app.js` into a sortable/filterable board with
  ECharts figures (two-pillar scatter with iso-Glass contours; CWR spread bars).
- `datasheet.html` — rendered from the committed `DATASHEET.md` (restyled; the
  integrity-rule and limitations sections are visually promoted by heading slug; a
  split-count donut is injected) plus the donut figure.
- `style.css` — the shared dark "instrument" theme.
- `app.js` — vanilla JS: count-up, the confidence dial, sort/filter/detail/copy on the
  board, and the ECharts charts. Degrades cleanly if absent or if the CDN is blocked.
- `leaderboard.json` — the verified leaderboard data (the vestigial `weights` field is
  never present).

The repo root also has a `Dockerfile` (multi-stage: render with `python:alpine` → serve
with `nginx:alpine` on **port 80**) and a `.dockerignore`.

## Build locally

```bash
python site/build_site.py            # writes site/dist/
```

Open `site/dist/index.html`, or serve it: `python -m http.server -d site/dist 8080`.
The build is deterministic — two runs are byte-identical.

## Build + serve with Docker

```bash
docker build -t glassbench-site .
docker run --rm -p 8080:80 glassbench-site   # http://localhost:8080
```

The image regenerates the output from the current Markdown + generator in its build
stage, so the committed `dist/` is a convenience snapshot, not the source of truth.

## Deploy

`.github/workflows/deploy.yml` pings a Coolify deploy webhook on every push to `main`
(Coolify builds the `Dockerfile` and serves the nginx static site). It needs two repo
secrets: `COOLIFY_APP_UUID` and `COOLIFY_TOKEN`.
